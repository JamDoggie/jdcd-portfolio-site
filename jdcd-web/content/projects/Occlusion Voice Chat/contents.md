[Occlusion Voice Chat](https://github.com/JamDoggie/Occlusion-voice-chat) was an older project of mine intended for use for a private Minecraft event I used to host (Ultra Hardcore Server.) It is an applications users can download and connect to a voice chat server with. It uses Avalonia for the UI. The features of this project are:

- **Completely server-side approach.** At the cost of some latency when moving around in-game, this means users do not need to modify their games. A server-side mod can be installed that handles authentication and routing user voice chat data.
- **Custom HRTF implementation.** This project has a neat implementation for HRTFs using pretty low level access to OpenAL APIs. This is pretty much as low-level as you can go while still having easy cross-platform support on desktop. HRTFs (Head-Related Transfer Functions) are useful for accurately portraying how sound travels through our ears. This makes it intuitive and easy for users to pinpoint where sounds are coming from naturally. [Steam Audio](https://valvesoftware.github.io/steam-audio/) is a popular library that I use these days that implements HRTFs natively. Google's [Resonance Audio](https://resonance-audio.github.io/resonance-audio/) is another library that implements HRTFs, and is a little more barebones.
